# スムストック物件追跡システム 提案書

## 目的

スムストックから取得した物件情報を時系列で追跡し、以下を実現する：

1. **物件の販売状況追跡** - どの物件が売れたか検出
2. **価格変動の記録** - 値下げ・値上げの履歴
3. **販売期間の分析** - 掲載から売却までの期間
4. **市場動向の可視化** - グラフと統計による洞察

## 国土交通省APIの活用可能性

### 利用可能なAPI

**不動産取引価格情報API (XIT001)** を使用することで、実際の成約価格データが取得可能：

- ✅ 実際の売買成約価格
- ✅ 取引時期（四半期ごと）
- ✅ 物件所在地（町丁目レベル）
- ✅ 建物面積、土地面積
- ✅ 建築年、構造

### 制約事項

#### 1. 物件の特定が困難
- MLITデータは個別物件を特定できない形式（プライバシー保護）
- スムストックデータとの完全なマッチングは不可能
- 面積・場所・時期での**推定マッチング**のみ可能

#### 2. タイムラグ
- 取引データは四半期ごとに公開
- 実際の成約から2〜6ヶ月遅れる
- リアルタイム追跡には使えない

#### 3. マッチング精度
- 同じ町丁目に複数の類似物件がある場合、特定困難
- 建物面積が完全一致しても別物件の可能性
- **参考情報**としての利用に限定

### 結論：段階的アプローチを推奨

## 推奨実装：2段階アプローチ

### 第1段階：スムストックデータのみでの追跡（確実・高価値）

**実装内容：**

```
1. 月次比較システム
   - 前月データと今月データを比較
   - 消えた物件 → 「販売終了」として記録
   - 価格変更を検出

2. 物件追跡データベース
   - property_id: ユニークID（所在地+面積でハッシュ生成）
   - first_seen: 初回掲載日
   - last_seen: 最終確認日
   - status: active / sold_presumed / unknown
   - price_history: [{date, price}]

3. 分析・可視化
   - 販売期間の分布
   - 価格変動パターン
   - エリア別・ハウスメーカー別の傾向
```

**メリット：**
- ✅ 確実なデータ（推測なし）
- ✅ リアルタイム性が高い
- ✅ 実装がシンプル
- ✅ ユーザーにとって価値が高い情報

**生成される洞察：**
- 平均販売期間：X日
- 価格変更率：Y%が値下げ
- 人気エリア：Z地域が最速で売れる
- ベストタイミング：掲載後N日で値下げ傾向

### 第2段階：MLIT APIとの照合（実験的・参考情報）

**実装内容：**

```
1. 消えた物件のリスト作成
   - status = sold_presumed の物件を抽出
   - 推定売却時期から3ヶ月後に照合開始

2. MLIT APIからデータ取得
   - 該当エリアの取引価格データ
   - 四半期単位で取得

3. マッチング候補の抽出
   - 建物面積が±5%以内
   - 土地面積が±5%以内
   - 所在地が同じ町丁目
   - 取引時期が推定時期の前後3ヶ月以内

4. 結果の表示
   - 「参考：類似物件の成約事例」として提示
   - 確実な同一物件ではないことを明記
   - 複数候補がある場合は全て表示
```

**メリット：**
- 市場価格との比較が可能
- 実際の成約価格の参考値
- データの信頼性向上

**デメリット：**
- マッチング精度に限界
- 四半期ごとのタイムラグ
- API利用料・レート制限の考慮必要

## まとめ

**✅ 推奨：第1段階を優先実装**

スムストックデータのみでの追跡システムは：
- 確実に実現可能
- 高い精度
- ユーザーにとって価値が高い
- 実装コストが低い

**⚠️ MLIT APIは補助的に活用**

国土交通省APIとの照合は：
- 参考情報として有用
- 完全なマッチングは期待しない
- Phase 3として後から追加

この2段階アプローチにより、確実に価値あるデータを提供しながら、段階的に機能を拡張できます。
